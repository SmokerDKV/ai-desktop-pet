# 📦 项目文件说明

本文档说明项目中各个文件的作用。

## 核心程序文件

| 文件 | 说明 |
|------|------|
| `deskpet.py` | 主程序，包含桌面宠物的所有核心功能 |
| `memory_manager.py` | 记忆管理模块，处理对话记录和长期记忆 |
| `personality_manager.py` | 性格管理模块，处理角色配置的保存和加载 |
| `personality_setting.py` | 性格设定界面，提供图形化配置界面 |

## 配置文件

| 文件 | 说明 | 是否上传 |
|------|------|----------|
| `配置文件/config.example.json` | 配置文件示例 | ✅ 上传 |
| `配置文件/config.json` | 实际配置文件（含 API Key） | ❌ 不上传 |
| `personality_config.json` | 性格配置文件 | ❌ 不上传 |

## 数据文件

| 文件/目录 | 说明 | 是否上传 |
|-----------|------|----------|
| `memory.db` | SQLite 数据库，存储对话记忆 | ❌ 不上传 |
| `人物图片/` | 用户上传的角色图片 | ❌ 不上传 |
| `配置文件/pet_images/` | 表情图片目录 | ❌ 不上传 |
| `配置文件/logs/` | 日志文件目录 | ❌ 不上传 |

## 脚本文件

| 文件 | 说明 |
|------|------|
| `启动.bat` | 一键启动脚本，自动安装依赖并运行 |
| `初始化.bat` | 依赖安装脚本 |
| `重置所有数据.bat` | 重置所有用户数据和配置 |
| `上传到GitHub.bat` | 快速上传到 GitHub 的辅助脚本 |

## 占位文件

| 文件 | 说明 |
|------|------|
| `人物图片/.gitkeep` | 保持目录结构的占位文件 |

## 首次运行时自动生成的文件

以下文件在首次运行程序时会自动生成：

- `配置文件/config.json` - 从 `config.example.json` 复制或通过界面创建
- `memory.db` - 首次对话时自动创建
- `personality_config.json` - 首次配置性格时创建

## 文件大小参考

- 核心程序：约 80KB
- 依赖安装后：约 50MB（包含 Python 库）
- 用户数据：
  - `memory.db`：随使用增长，通常 < 10MB
  - 图片文件：每张 1-5MB，共约 50MB

## 备份建议

建议定期备份以下文件：

1. `配置文件/config.json` - API Key 和窗口配置
2. `personality_config.json` - 角色性格配置
3. `memory.db` - 对话记忆数据库
4. `人物图片/` - 自定义角色图片

## 清理说明

如需清理项目，可以安全删除：

- `__pycache__/` - Python 缓存，会自动重建
- `配置文件/logs/*.log` - 日志文件
- `memory.db` - 对话记忆（如需重置）

---

💡 提示：使用 `重置所有数据.bat` 可以一键清理所有用户数据。
